## Como subir em outro computador

## 1. Clonar o repositorio

```bash
git clone https://github.com/davidpazuello-sudo/sales-ops-backup.git
cd sales-ops-backup
```

## 2. Instalar dependencias

```bash
npm install
```

## 3. Trazer variaveis de ambiente

Opcao recomendada se a Vercel estiver acessivel:

```bash
vercel pull --yes --environment=production
vercel pull --yes --environment=preview
```

Ou montar manualmente um `.env.local` com base em:

- `.env.example`
- `.env.development.example`
- `.env.staging.example`
- `.env.production.example`

## 4. Variaveis mais importantes

Nao commitar segredos. Em outro computador, garantir pelo menos:

- `NEXT_PUBLIC_APP_URL`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `SUPER_ADMIN_EMAILS`
- `CORPORATE_EMAIL_DOMAINS`
- `HUBSPOT_ACCESS_TOKEN_STAGING`
- `HUBSPOT_ACCESS_TOKEN_PRODUCTION`
- `HUBSPOT_CLIENT_SECRET_STAGING` se usar webhook em staging
- `HUBSPOT_CLIENT_SECRET_PRODUCTION` se usar webhook em producao

## 5. Validar a maquina

```bash
npm run verify
```

Se precisar validar tambem os fluxos E2E:

```bash
npm run e2e
```

## 6. Rodar local

```bash
npm run dev
```

## 7. Se precisar operar migrations

Usar Supabase CLI com:

- `SUPABASE_ACCESS_TOKEN`
- `project ref`: `ilbgvuveznlxdnypmzql`
- senha do banco remoto guardada fora do repo

Fluxo:

```bash
supabase link --project-ref ilbgvuveznlxdnypmzql --password "<senha>"
supabase db push
```
